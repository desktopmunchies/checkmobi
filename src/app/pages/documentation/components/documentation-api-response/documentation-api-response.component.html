API Response

All CheckMobi APIs return a response in json format. The API returns one of the following HTTP status codes.

  • 200 – Success. The response has content
  • 204 – Success. No content available
  • 400 – A parameter is missing or is invalid
  • 401 – Authentication failed (invalid token or expired)
  • 403 – Forbidden (you don't have access to that resource)
  • 404 – Resource cannot be found
  • 500 – Server error

Available error codes

In case you receive one of the above HTTP status codes that indicates an error (400, 401) you will receive in the response body one of the below error codes and the description to help you to debug the route of the problem.

Code Description
1 Invalid or expired API secret key
2 Invalid phone number. Phone number should be provided into E.164 format
3 Invalid request id
4 Invalid validation type. Allowed (sms, ivr, reverse_cli)
5 Insufficient funds
6 Insufficient CLI validation count
7 Invalid request payload
8 Validation method not available in this country
9 URL provided to receive notifications about validations is invalid
10 Invalid event payload
11 Restricted account due to suspicious activity
12 Your account exceeded one or more limitations
13 Destination is blocked due to application restrictions
14 Number temporary blocked because of too many attempts
