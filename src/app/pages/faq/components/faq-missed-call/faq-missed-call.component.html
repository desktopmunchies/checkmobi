Missed Call

Can I implement just reverse CLI using your sdk?
You can, but we recommend implementing SMS as fallback for number verification. The reason is that on some destinations Caller ID is not sent correctly. You have similar problems with SMS when it does not reach or has 1h delays. The best way to do this is :
  1. Set Missed call (Reverse caller ID) as primary method of verification.
  2. Use SMS or/and IVR call as fallback if Missed Call fails.

Why do I get "Insufficient funds" when trying "Missed Call" if verification is free?
Although a successful Missed Call verification does not cost anything, there are reasons when a Missed Call can cost you:
  1. If your user enters a wrong number and we call someone else, and that person answers the call.
  2. In case your user answers the missed call fast enough.
  3. If the call is rejected from the phone side and user has voice mail activated on call reject. (That is why it is very important when you implement Missed Call to always call our API to hangup the call from the server. Never from the client directly)

If a missed call is answered we will charge you the rate for 1 minute of calling on that destination. See rates here.

Because we get charged, we need to charge you when these rare situations happen. That is why you need to have credit available in order to use Missed Call.

Does the call come from the same number or random numbers?
Call comes from random numbers. We have many numbers that we change periodically.
What is your Missed Call coverage?
Missed Call coverage is global. We can use missed call in all countries where we have ability to send a call. To check if we support a certain country, see this link here.
Missed Calls do not arrive on the device, what can be the problem?
Make sure your user entered correctly his number. If this is the case, then it could be a problem of delay on the network side from us sending the call to reaching your user. This delay can have a few seconds.
There is a setting in "General Settings" section within your application called: "Missed Call ring time". This is the time it passes before we hangup the call, so your users don't have time to answer it. Try increasing it and see if calls reach your desired destination.
Calls arrive with a wrong ID ? How can I fix this?
If you correctly implemented missed call verification it means you have 2 things:
  1. SMS is set as fallback when Missed Call does not send correct Caller ID.
  2. You can turn on/off Missed Call on a certain destination.
In this case, just turn off Missed Call for that destination and open a ticket on support[@]checkmobi.com
